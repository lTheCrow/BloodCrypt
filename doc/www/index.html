Sobre el proyecto

El proyecto BloodCrypt es un ransomware que va a estar hecho en C destinado a computadoras con Windows instalado. El objetivo de esto no es ganar dinero, si no, aprender a desarrollar malware utilizando las herramientas que utilizaria un desarrollador de malware en la vida real.

Quienes podran participar en el proyecto

Ya que es un proyecto de aprendizaje no voy a poner muchos requisitos. Basicamente los que quieran participar deben tener lo siguiente:

  • Tener una computadora con procesador de arquitectura Intel x86-64 o AMD64
  • Tener alguna version de Windows 7/8/10/11, ya que el malware que haremos estara hecho en el
  • Es recomendable poder ejecutar una maquina virtual con otro Windows
  • Tener un nivel de ingles B1 o superior, basicamente que pueda leer textos en ingles (aunque le cueste un poco)
  • Saber el lenguaje C en un nivel muy basico, que sepa hacer un hello world, usar variables, operaciones aritmetico-logicas y mas caracteristicas basicas del lenguaje
  • Tener un interes enorme sobre algun tema relacionado a:
    • Ingenieria Inversa
    • Analisis y Desarrollo de Malware
    • Explotacion Binaria y Desarrollo de Exploits a bajo nivel
    • Evasion de Antivirus
    • Programacion en C y lenguaje Ensamblador
    • Desarrollo de Sistemas Operativos
  • Tener ganas de aprender con el proyecto. Este requisito no es de relleno, de verdad les pido que si piden para participar sean serios y aun si no saben mucho que tengan la voluntad de aprender todos los dias aunque sea un poco para despues aportar al proyecto, con eso me vale

Caracteristicas de BloodCrypt

BloodCrypt v1.0

Para ya fijar varios objetivos, voy a poner las caracteristicas que debera tener la version 1 del ransomware. Cualquier modificacion o idea que quieran hacer pueden notificarme, mis datos de contacto estan abajo :).

  • Ejecutarse en el sistema operativo Windows bajo procesadores de arquitectura Intel x86-64 y AMD64
  • Crear una clave aleatoria para encriptar con AES
  • Encriptar archivos utilizando el algoritmo AES256
  • Al encriptar un archivo se escribira en un archivo el directorio de ese archivo
  • Esa clave aleatoria se cifrara con algun algoritmo ya que despues se mostrara a la victima
  • Al encriptar los archivos, se le mostrara una wallet de bitcoin y la clave cifrada
  • Al recibir el pago, la victima enviaria por correo la clave cifrada para que con un programa de nuestro dominio descifre nuestro algoritmo y le respondamos el correo con la clave
  • La victima abriria otro archivo ejecutable donde pondria la clave, este se encargara de desencriptar los archivos registrados en el archivo
